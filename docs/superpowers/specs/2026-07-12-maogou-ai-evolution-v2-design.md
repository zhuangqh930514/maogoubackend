# 猫狗智投 AI 进化 V2 设计规范

## 1. 目标

猫狗智投 AI 进化 V2 的目标是建立可复现、可验证、可晋级、可回滚的 A 股投研学习闭环。系统不承诺胜率单调上升，而是保证只有通过样本外验证的策略才能影响用户结果，退化策略能够被识别和回滚。

用户的日常路径保持简单：维护自选股和持仓，查看每日投研与投研日报。样本、因子、实验、回测和模型版本进入专家工作台。

## 2. 核心原则

1. 预测时点之后的数据不得进入样本。
2. 样本、因子、预测和标签均不可原地改写。
3. 低置信度只代表无法判断，不能自动解释为看空。
4. LLM 负责解释和资讯理解，数值模型负责评分和排序。
5. 旧版学习数据标记为 `LEGACY`，不进入 V2 策略训练。
6. 新策略先影子运行，满足晋级门槛后才可成为 Champion。
7. 每个用户可以没有推荐结果，禁止为了展示而制造推荐。
8. 所有审计链路使用真实外键；必需父记录不得缺失，可选父记录使用 `NULL`，禁止用 `DEFAULT 0` 伪造引用。

## 3. 目标数据流

```text
行情/财务/板块/资讯
  -> 数据批次与质量校验
  -> 可交易股票池
  -> Point-in-time 样本
  -> 版本化因子
  -> 规则基线/排序模型
  -> 不可变预测
  -> 每日投研 + LLM 解释
  -> 版本化交易日历 + T+1/T+3/T+5 标签 + 成本证据
  -> 训练数据集血缘 + Walk-forward + 组合回测
  -> Champion/Challenger 同样本影子评估
  -> 治理事件 + Champion 晋级或回滚
  -> 每日投研日报
```

## 4. V1 隔离策略

- `AiEvolutionServiceImpl` 不再删除或覆盖新版学习统计。
- `ai_factor_stat` 仅保留兼容展示，V2 使用独立表。
- 旧版策略可以查看和手工回滚，但不能自动晋级。
- 旧版回测和模型评测显示 `LEGACY_METRIC`，不能作为上线证据。
- V1 报告仍可展示，但只有绑定 V2 预测的报告进入 V2 模型评测。

## 5. V2 数据模型

### 5.1 `ai_data_batch`

记录一次数据采集批次：交易日、阶段、来源状态、行情/K线/财务/板块/资讯截至时间、质量分、失败原因和完成时间。

### 5.2 `ai_sample_v2`

保存不可变样本：用户、股票、交易日、阶段、`as_of_time`、数据批次、股票池版本、市场环境、板块、可交易状态、质量状态、特征快照、特征版本。

唯一键包含用户、股票、阶段、`as_of_time`、股票池版本，不允许同一时点覆盖。

### 5.3 `ai_factor_value_v2`

保存因子版本、原始值、标准化值、方向、是否缺失、缺失原因和证据。连续因子保留符号，不使用统一绝对值归一化。

### 5.4 `ai_prediction_v2`

保存股票、交易日、样本阶段、推理模式、输入指纹、策略版本、可选模型版本、预测周期、预期收益、预期超额收益、上涨/下跌概率、风险分、校准置信度、截面排名、动作、主动放弃原因和解释快照。记录一经创建不可更新。

除调用方幂等键外，业务唯一键由用户、股票、交易日、阶段、周期、策略发布、推理模式和输入指纹组成。策略发布是必需父记录；纯规则推理没有模型版本时，`model_version_id` 为 `NULL`。

### 5.5 `ai_label_v2`

保存真实入场/出场日历记录和价格、日历版本、输入指纹、交易可执行状态、原始收益、净收益、指数收益、板块收益、超额收益、MFE、MAE、方向与目标命中、标签版本、成熟时间和首次验证时间。待成熟标签的日历关联允许为 `NULL`，成熟后必须指向当时使用的版本化交易日历记录。

### 5.6 `ai_factor_performance_v2`

按因子版本、预测周期、市场环境、滚动窗口统计样本数、RankIC、命中率、平均超额收益、回撤、Wilson 下界、稳定性和漂移状态。窗口唯一键同时包含 `window_start_date` 与 `window_end_date`，不能让不同起点的窗口相互覆盖。

### 5.7 `ai_model_version` 与 `ai_strategy_release`

模型版本必须关联训练数据集，记录训练器版本、随机种子、特征版本、训练/验证/测试窗口、产物与特征清单校验和、校准方式和样本外指标。策略发布记录 Champion/Challenger、影子运行、晋级证据和回滚原因；纯规则策略的模型关联为 `NULL`。

`active_champion_guard` 生成列与唯一键在数据库层保证每个用户最多一个 `release_role=CHAMPION AND status=ACTIVE` 的策略，服务层状态机不能绕过该约束。

### 5.8 `ai_pipeline_run` 与 `ai_pipeline_step`

记录流水线级幂等键、输入指纹、数据批次/策略/模型关联、步骤状态、重试次数、输入输出数量、检查点、输出指纹、错误和耗时。步骤失败可单独重试，不能重复创建预测。

### 5.9 `ai_research_daily_report`

每天每用户可保留多个不可变版本，保存摘要、市场环境、推荐/观察/回避数量、持仓风险、关键因子、策略表现、数据质量、任务状态、结构化内容和生成时间。调用幂等键防止重复生成，`current_report_guard` 生成列唯一键保证每用户每交易日最多一份 `is_current=1` 日报，`supersedes_report_id` 保留替代链。

### 5.10 `ai_trading_calendar` 与 `ai_label_cost_evidence`

交易日历按市场、日期、`calendar_version` 唯一，保存开闭市时间、前后交易日、来源截至时间和来源指纹。标签成本证据按标签和成本模型版本唯一，保存数量、买卖名义金额、佣金/印花税/过户费率、滑点、逐项金额、总成本和原始证据 JSON。

### 5.11 `ai_training_dataset` 与 `ai_training_dataset_item`

数据集保存用途、特征/标签/日历版本、可见性截止时间、训练/验证/测试窗口、选择规则、来源查询、产物校验和与血缘指纹。明细逐行关联不可变样本和已成熟标签，保存 split、顺序、样本时点、标签可用时间以及特征/标签指纹，支持证明训练时没有看到未来数据。

### 5.12 Walk-forward 与组合回测

`ai_walk_forward_run` 保存数据集、策略/模型、purge/embargo、随机种子、输入指纹和聚合指标；`ai_walk_forward_fold` 保存不重叠的训练/验证/测试窗口；`ai_walk_forward_baseline` 保存指数、等权池、动量和 Champion 基线。

组合回测使用 `ai_portfolio_backtest_run`、`ai_portfolio_backtest_daily`、`ai_portfolio_backtest_trade` 与 `ai_portfolio_backtest_position`。run 固化引擎版本、配置/输入指纹和随机种子；daily 保存复利净值；trade 保存执行与成本；position 保存每日仓位、权重、盈亏和单票贡献。

### 5.13 影子评估、漂移与治理事件

`ai_shadow_evaluation` 保存 Champion/Challenger 的窗口级覆盖率、校准、收益、回撤、特征漂移和晋级判断；`ai_shadow_evaluation_item` 对同一不可变样本保存双方预测配对，影子结果不能进入用户动作。

`ai_drift_event` 用事件指纹幂等记录因子、模型、策略或影子窗口的指标、阈值、严重度、状态和证据。`ai_strategy_governance_event` 记录晋级、拒绝、回滚和人工确认，并可关联 Walk-forward、回测和影子评估证据。

### 5.14 完整性与查询契约

- 样本到批次、因子到样本、预测到样本/策略/模型、标签到预测/样本/日历，以及 run/item 明细均声明外键。
- 可选模型、证据 run 和替代记录使用 `BIGINT NULL`；所有 `*_id` 均禁止 `DEFAULT 0`。
- 主要读取路径按用户/交易日/阶段/状态、策略/窗口、run/日期和 current 状态建立组合索引。
- `schema.sql` 只保留一个 V2 区块，且该区块与 `20260712_ai_evolution_v2.sql` 完全一致。

## 6. 标签口径

- `AFTER_CLOSE` 预测以下一交易日开盘价作为默认入场价。
- 第一版生成 T+1、T+3、T+5 三套标签。
- 买入信号使用净超额收益和风险约束评价。
- 回避信号评价是否成功规避下跌。
- `WATCH` 作为主动放弃样本，不计入普通方向胜率。
- 停牌、涨停买不进、跌停卖不出、ST、新股和低流动性分别记录状态。
- 净收益必须能追溯到成本模型版本、费率、滑点、逐项金额和来源指纹。
- 标签创建后不覆盖；需要修正时增加 `label_version`。

## 7. 最终决策语义

- `RECOMMEND`：模型信号为正、进入 Top K、数据新鲜、风险门控通过、策略状态可用。
- `WATCH`：模型分歧、历史可靠性低、数据部分缺失或置信度不足。
- `AVOID`：当前模型明确看空、高风险或不可交易。
- `UNAVAILABLE`：核心数据过期或缺失，不输出投研动作。

历史命中率只影响置信等级。LLM 的 `WATCH` 不能因为历史命中率低被转换成 `REDUCE`。

## 8. 实验、回测与策略治理

- 按时间执行 Purged Walk-forward，禁止随机切分。
- 每个交易日独立选择 Top K。
- 使用沪深300、股票池等权、简单动量和当前 Champion 作为真实基线。
- 回测计算组合净值、仓位、手续费、印花税、滑点、T+1、停牌和涨跌停。
- Walk-forward、回测和影子评估都必须固化输入指纹；同一 run key 不得产生第二份结果。
- Challenger 至少覆盖 60 个交易日、1000 条预测和 200 笔可执行交易。
- 至少三个 Walk-forward 窗口，Top K 超额收益置信下界大于零。
- 最大回撤不得超过 Champion 的 1.1 倍，单票利润贡献不得超过 15%。
- 初期晋级必须人工确认；回撤、漂移和数据异常触发自动回滚。
- 每次晋级、拒绝和回滚都写治理事件；数据库拒绝同一用户出现两个 active Champion。

## 9. 自动化任务

每日任务：数据同步、质量校验、成熟标签验证、因子监控、样本构建、预测排序、重点股票 LLM 报告、每日投研快照、投研日报。流水线 run 关联数据批次、策略和模型，步骤持久化检查点以支持从失败点恢复。

周任务：默认周五 18:00 执行 60 日滚动因子表现、前后 60 日漂移、Purged Walk-forward、组合回测和 Challenger 评估。行情基准固定为沪深 300 `000300.SH`；K 线覆盖或成熟标签不足时记录 `SKIPPED`，不得生成伪回测。

月任务：默认每月 1 日 19:00，在 `LABEL_V2.2` 的 T+3 成熟样本达标后训练模型、校准概率、导出 ONNX 并生成候选模型版本。旧标签以及 T+1/T+5 标签不进入当前 T+3 排序模型，同一 `sampleId` 只能出现一次；未通过样本外门槛的模型保持 `CANDIDATE`，只有 `VALIDATED` 模型可创建 `SHADOW` Challenger，且永不自动晋级 Champion。单机阶段不引入 Redis；只有多实例调度时再增加分布式锁或消息队列。

## 10. 前端信息架构

`AI分析` 菜单顺序：

1. 每日投研
2. 投研日报
3. 分析报告
4. 研究实验室

普通用户不直接操作样本、因子、实验、回测和模型评测。投研日报展示当天结论、与昨日变化、推荐关注、风险回避、持仓风险、关键因子、历史证据、数据新鲜度、策略版本和自动任务状态。

## 11. 错误与降级

- 核心行情不可用：停止预测，日报标记 `DATA_UNAVAILABLE`。
- 资讯不可用：继续数值预测，明确标记“无最新资讯”。
- LLM 不可用：保留数值结果，日报使用结构化模板生成。
- 单只股票失败：隔离记录，不影响其他股票。
- 流水线中断：从失败步骤继续，不重复已完成步骤。
- 模型漂移或策略退化：降级为 Champion 或规则基线。

## 12. 验收标准

- 未来数据扰动不改变历史预测。
- 同一幂等键重复执行不产生重复样本、预测、标签或日报。
- 预测业务键、因子窗口起止边界、训练血缘和所有实验输入指纹可审计。
- 数据不新鲜时没有推荐结果。
- `WATCH + HISTORY_WEAK` 最终仍为 `WATCH`。
- 回测 Top K 数量按交易日约束，组合净值可复现。
- 自动流水线结束后必有一份日报；失败时也生成异常日报。
- 每个用户最多一个 active Champion；每用户每交易日最多一份 current 日报。
- 增量脚本不存在 `*_id BIGINT ... DEFAULT 0`，核心关联均有外键和查询索引。
- 全量 schema 的 V2 区块只出现一次，并与增量脚本保持一致。
- 每日投研和日报读取接口 P95 小于 3 秒。
- 后端单测、集成测试、前端构建和关键浏览器流程全部通过。
